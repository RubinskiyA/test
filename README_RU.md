# ByeByeDPI для iOS

Приложение для обхода DPI (Deep Packet Inspection) на устройствах iOS. Является адаптацией популярного Android-приложения [ByeByeDPI](https://github.com/romanvht/ByeByeDPI).

## 📱 Возможности

- **Обход DPI**: Эффективный обход систем глубокой проверки пакетов
- **Авто-определение стратегии**: Автоматический подбор оптимальных параметров для вашего провайдера
- **Гибкие настройки**: Ручная настройка всех параметров обхода (TTL, фальшивые пакеты, разделение и др.)
- **Локальная работа**: Весь трафик обрабатывается локально на устройстве, без внешних серверов
- **Open Source**: Полностью открытый исходный код на основе оригинального byedpi

## 🔧 Как это работает

Приложение использует:
1. **Network Extension (Packet Tunnel)** - для перехвата всего сетевого трафика устройства
2. **byedpi** - движок обхода DPI (написан на C, кроссплатформенный)
3. **hev-socks5-tunnel** - библиотека для создания SOCKS5 туннеля (официально поддерживает iOS)

Трафик перенаправляется через локальный туннель, где byedpi модифицирует пакеты согласно выбранной стратегии, что позволяет обходить блокировки провайдера.

## 🚀 Быстрый старт

### Требования
- macOS 12+ с установленным Xcode 14+
- iOS устройство с iOS 14.0+ (для тестирования на реальном устройстве)
- Apple Developer аккаунт (бесплатного достаточно для личной разработки)

### Установка и сборка

1. **Клонируйте репозиторий:**
   ```bash
   git clone https://github.com/YOUR_USERNAME/ByeByeDPI-iOS.git
   cd ByeByeDPI-iOS
   ```

2. **Скомпилируйте byedpi:**
   ```bash
   chmod +x Scripts/build_byedpi.sh
   ./Scripts/build_byedpi.sh
   ```
   Скрипт автоматически загрузит и скомпилирует библиотеку byedpi для iOS.

3. **Откройте проект в Xcode:**
   ```bash
   open ByeByeDPI.xcodeproj
   ```

4. **Настройте подписку:**
   - Выберите целевое устройство (ваш iPhone/iPad)
   - В настройках проекта (Project → Signing & Capabilities) выберите вашу Team
   - Повторите для обоих targets: `ByeByeDPI` и `PacketTunnel`

5. **Добавьте необходимые capabilities:**
   - Для основного приложения: `App Groups` (создайте группу вида `group.com.yourname.byebyedpi`)
   - Для PacketTunnel: `App Groups` (ту же самую), `Network Extensions`

6. **Запустите проект:**
   - Нажмите `Cmd+R` или кнопку Play в Xcode
   - На устройстве разрешите установку непроверенного приложения (Настройки → Основные → VPN и управление устройством)

## 📖 Использование

### Подключение
1. Откройте приложение
2. Нажмите большую кнопку подключения в центре экрана
3. При первом подключении система запросит разрешение на добавление VPN конфигурации - нажмите "Разрешить"
4. Статус изменится на "Подключено"

### Авто-определение лучшей стратегии
1. Перейдите в раздел "Настройки" (Settings)
2. Нажмите кнопку **"Авто-определение"** (Auto-detect Best Strategy)
3. Дождитесь завершения теста (может занять 30-60 секунд)
4. Приложение покажет лучшую найденную стратегию
5. Нажмите "Применить" для использования этих настроек

### Ручная настройка
В разделе настроек вы можете вручную изменить:
- **Desync TTL**: Значение десинхронизации TTL
- **Fake Packets**: Тип и количество фальшивых пакетов
- **Packet Split**: Разделение пакетов по размеру
- **HTTP Method**: Метод подмены HTTP заголовков
- И другие параметры

## 🔍 Технические детали

### Архитектура
```
[SwiftUI UI] 
    ↓
[VPNManager] ←→ [SettingsManager]
    ↓
[PacketTunnel Provider] (Network Extension)
    ↓
[ByeDpiProxy] (Swift ↔ C Bridge)
    ↓
[byedpi] (C Library) + [hev-socks5-tunnel]
```

### Режимы работы
- **VPN режим**: Полный перехват трафика через NEPacketTunnelProvider
- **Тест стратегий**: Прямое подключение через сокет для измерения задержек

### Безопасность
- Приложение не отправляет данные на внешние серверы
- Весь код открыт для аудита
- Используется стандартный iOS Network Extension API

## ⚠️ Важные замечания

1. **App Store**: Публикация в App Store может быть ограничена из-за функционала обхода блокировок. Рекомендуется распространять через TestFlight или личную подпись.

2. **Фоновая работа**: iOS ограничивает работу в фоне. Приложение использует стандартные механизмы Network Extension для поддержания соединения.

3. **Производительность**: Network Extension имеет лимит памяти (~15-20MB). При сложных стратегиях возможна повышенная нагрузка на CPU.

4. **Hotspot**: Обход DPI для режима модема (Personal Hotspot) может не работать из-за ограничений iOS.

## 🛠 Разработка

### Добавление новых стратегий
Стратегии определяются в файле `StrategyTester.swift`. Для добавления новой:
1. Добавьте параметры в структуру `StrategyParams`
2. Обновите массив `defaultStrategies`
3. Протестируйте на различных провайдерах

### Отладка
- Логи основного приложения: Console.app на Mac
- Логи Network Extension: требуют отдельной схемы отладки в Xcode
- Используйте `os_log` для вывода отладочной информации

### Сборка релизной версии
```bash
xcodebuild -scheme ByeByeDPI -configuration Release archive
```

## ❓ FAQ

**Q: Работает ли с Wi-Fi и мобильной сетью?**  
A: Да, приложение работает с любыми типами соединений.

**Q: Нужно ли платить за подписку?**  
A: Нет, для личной разработки достаточно бесплатного Apple ID.

**Q: Почему не работает на симуляторе?**  
A: Network Extension требует реального устройства для тестирования.

**Q: Как удалить VPN профиль?**  
A: Настройки → Основные → VPN и управление устройством → Удалить конфигурацию.

**Q: Авто-определение занимает слишком много времени**  
A: Это нормально. Тестируется 25+ комбинаций параметров. Результат сохраняется для будущего использования.
