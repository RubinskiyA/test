# Функция авто-определения лучшей стратегии (Auto-detect Best Strategy)

## Обзор

Функция автоматического тестирования стратегий обхода DPI, аналогичная той, что есть в Android-версии ByeByeDPI. Эта функция тестирует различные комбинации параметров byedpi и определяет оптимальную стратегию для вашего интернет-провайдера.

## Реализованные файлы

### 1. `StrategyTester.swift`
Класс для управления процессом тестирования стратегий:

**Основные возможности:**
- Автоматический перебор 25+ стратегий обхода
- Измерение задержки для каждой рабочей стратегии
- Определение лучшей стратегии по минимальной задержке
- Progress tracking во время тестирования
- Сохранение результатов в настройки

**Тестируемые стратегии:**
1. **Базовые** - без модификаций пакетов
2. **Desync TTL** - десинхронизация с TTL=1,2,3,5,10
3. **Fake Packets** - фальшивые TLS/HTTP пакеты
4. **Packet Split** - разделение пакетов на разных позициях
5. **OOO (Out-of-Order)** - пакеты вне порядка
6. **Комбинированные** - сочетание нескольких методов
7. **SNI манипуляции** - отключение/подмена SNI

### 2. Обновленный `SettingsView.swift`
UI для функции авто-определения:

**Элементы интерфейса:**

## Как это работает

### Алгоритм тестирования:

```swift
1. Пользователь нажимает "Auto-detect Best Strategy"
2. StrategyTester начинает перебор стратегий:
   ├── Для каждой стратегии:
   │   ├── Создает аргументы для byedpi
   │   ├── Вызывает test_byedpi_strategy()
   │   ├── Замеряет время подключения
   │   └── Сохраняет результат (успех/неудача + latency)
   │
   ├── Если успех → сравнивает с текущей лучшей
   └── Обновляет UI с прогрессом
3. После завершения:
   ├── Показывает лучшую стратегию
   ├── Предлагает применить её
   └── Позволяет просмотреть все результаты
```

### Пример использования:

```swift
// В SettingsView
@StateObject private var strategyTester = StrategyTester()

// Запуск тестирования
private func startStrategyTesting() {
    Task {
        await strategyTester.startTesting()
    }
}

// Применение лучшей стратегии
Button("Apply This Strategy") {
    strategyTester.saveBestStrategy(to: settingsManager)
}
```

    // Выполняет тестовое подключение

## Отличия от Android версии

| Аспект | Android | iOS |
|--------|---------|-----|
| UI Technology | Jetpack Compose | SwiftUI |
| Background execution | Foreground Service | Limited (тест быстрый) |
| Network access | VpnService | NEPacketTunnelProvider |
| State management | ViewModel | ObservableObject |
| Async pattern | Coroutines | async/await |

## Преимущества iOS реализации

✅ **Быстрее**: Прямой вызов C функций без JNI overhead  
✅ **Реактивный UI**: Swift UI автоматически обновляется при изменении state  
✅ **Типобезопасность**: Swift compiler проверяет типы на этапе компиляции  
✅ **Интеграция**: Нативная интеграция с iOS настройками через UserDefaults  

## Требования

- iOS 14.0+
- Xcode 13.0+
- Swift 5.5+
- byedpi исходный код (клонируется автоматически при сборке)

## Примечания

⚠️ **Важно**: Функция тестирования требует активного интернет-соединения  
⚠️ **Время выполнения**: Полный тест занимает 30-120 секунд в зависимости от количества стратегий  
⚠️ **Battery**: Процесс тестирования может потреблять значительный заряд батареи  
⚠️ **Network Extension**: Для полноценного тестирования может потребоваться активация VPN
